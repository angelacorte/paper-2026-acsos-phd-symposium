\begin{abstract}
Swarm robotic systems are increasingly relevant for missions in which many heterogeneous devices must operate in dynamic and potentially safety-critical environments.
%
Programming such systems remains challenging:
developers need to express objectives at the collective level,
while their execution is distributed across devices that may move,
fail, join or leave the system,
without a centralized point of coordination.
%
\acl{ac} addresses this issue by providing macroprogramming abstractions for specifying resilient and self-organizing collective behaviors.
%
However,
current \acl{ac} applications typically execute a single collective program and offer limited support for runtime adaptation,
preemption, concurrent applications, and safety guarantees during transient phases.

This research investigates the foundations of a \acl{cros} built on \acl{ac}:
a middleware layer that lifts ROS-inspired capabilities from individual robots to the collective level.
%
The paper outlines the main requirements for such a system and reports ongoing contributions on morphogenetic formation control,
runtime path replanning, self-stabilizing consensus, collective sensing, and safety guarantees.
%
These results provide initial building blocks towards a collective operating layer for swarm robotics,
with future work focusing on safe runtime adaptation and support for multiple concurrent collective processes.
\end{abstract}

\begin{IEEEkeywords}
aggregate computing, distributed systems, self-organizing systems, swarm robotics
\end{IEEEkeywords}

\section{Overall Problem and Motivation}\label{sec:overall-problem-and-motivation}
Swarm robotic systems are characterized by the presence of a large number of heterogeneous devices,
which are mobile and resource-constrained.
%
An example is a network of surveillance drones,
which have to coordinate with each other to monitor a given area,
while maintaining a desired swarm formation, avoiding collisions and respecting no-fly zones
and possibly allowing law enforcement to intervene in case of emergency.
%
Environmental monitoring systems and search-and-rescue operations can also benefit from swarm robotics,
such as wildfire monitoring and disaster response.

Such devices must coordinate with each other to achieve global objectives,
while adapting to changes in the environment and in the network topology,
possibly without a centralized point of coordination,
which can introduce a single point of failure and limit the scalability of the system.
%
These systems are often deployed in dynamic and unpredictable environments,
where devices may fail or be added/removed at any time.

The design of such systems is challenging,
as it requires ensuring that the system can autonomously adapt to changes in the environment and in the network topology,
while still achieving its goals.

Swarm robotics studies decentralized collective behavior~\cite{brambilla2013swarm},
while ROS~2~\cite{ros} provides middleware abstractions mainly centered on individual robots.
%
Swarm programming languages such as Buzz~\cite{DBLP:journals/software/PinciroliB16} support the expression of collective behavior,
but provide limited runtime support for its management and adaptation.
%
This work investigates how aggregate computing can provide such support,
enabling collective robotic applications to be executed,
adapted,
preempted,
monitored,
and constrained through decentralized interactions.

A possible approach is through macroprogramming~\cite{casadei22},
which allows developers to focus on the desired global behavior rather than on the individual behavior of each device.
%
The \ac{ac} paradigm~\cite{DBLP:journals/computer/BealPV15} builds on this idea,
providing a programming model for expressing collective-level behavior through the composition of \emph{computational fields}~\cite{JLAMP2019},
a distributed data structure that maps devices to their values,
and that can be manipulated through a set of high-level operators.
%
By design,
\ac{ac} is resilient to failures and scalable to large numbers of devices,
as it relies on local interactions and \emph{self-organization} to achieve global objectives.
%
Over the years, reusable building blocks~\cite{DBLP:journals/tomacs/ViroliABDP18}
supporting self-stabilizing~\cite{DBLP:journals/cacm/Dijkstra74} \ac{ac} behaviors
have been proposed and later used in high-level libraries such as MacroSwarm~\cite{DBLP:journals/lmcs/AguzziCV25},
which supports pattern formation in swarm robotics.
%%
%Moreover,
%a general formal framework for the specification of collective computational processes has been proposed~\cite{EAAI2020-processes},
%which are temporary distributed computations carried out by dynamic groups of devices,
%as well as a set of algorithms for efficient propagation and termination of such concurrent execution processes~\cite{DBLP:journals/fgcs/AudritoCT24}.

However,
typical \ac{ac} applications run one complex algorithm on each device,
without preemption,
without explicit lifecycle management for multiple collective applications,
and with limited support for dynamically changing the behavior of the system at runtime.
%
In the context of swarm robotics,
this is a limitation,
as multiple applications may need to run concurrently on the same devices,
and authorized operators (e.g., law enforcement) may need to stop or change the behavior of the system at runtime in case of emergency.
%
Moreover,
\ac{ac} relies on the \emph{eventual} consistency of the system to achieve self-stabilization,
which may not be sufficient in safety-critical scenarios,
where a temporary inconsistency may lead to unsafe situations,
such as surveillance drones colliding with each other or entering restricted areas.

\section{Objectives and Contributions}\label{sec:objectives-and-contributions}
% todo Obiettivo -> presentare un sistema operativo collettivo per gli sciami basato su macro-programmazione
% todo Perche? -> paradigma con astrazioni globali, dec, ..

This research investigates how \ac{ac} can support the development of a ROS-like~\cite{ros} \ac{cros},
providing reusable mechanisms and self-organizing building blocks for defining and executing collective robotic behavior.
%
The envisioned system should adapt autonomously to environmental and topological changes while preserving its collective objectives.

Its main requirements include
\begin{inlinelist}
    \item concurrent execution of multiple collective applications;
    \item runtime adaptation and preemption,
    allowing authorized operators to change the system behavior when required;
    \item safety enforcement during transient and stabilization phases;
    \item collective sensing through heterogeneous devices;
    \item collective motion and formation control;
    \item consensus and communication between processes operating in different spatial regions.
\end{inlinelist}

The contributions developed so far address these requirements through mechanisms for collective formation \emph{(v)},
task replanning \emph{(ii)},
distributed consensus \emph{(vi)},
collective sensing \emph{(iv)},
and runtime safety \emph{(iii)},
which constitute initial building blocks toward the envisioned \ac{cros}.

\section{Methods and Results}\label{sec:methods-and-results}

The research follows an incremental methodology,
investigating \ac{cros} capabilities through representative collective-robotics problems.
%
For each problem,
an \ac{ac}-based decentralized solution is designed,
evaluated under failures,
mobility,
and topology changes,
and generalized into reusable mechanisms.

% per prima cosa dovevamo scegliere un linguaggio,
% negli anni ne hanno fatti tot, abbiamo scelto ck perche portabile, multiplatform, offre una libreria ergonomica

Among existing \ac{ac} languages~\cite{DBLP:journals/tocl/AudritoVDPB19,DBLP:journals/softx/CasadeiVAP22,DBLP:journals/corr/abs-2603-29999},
we selected Collektive~\cite{DBLP:conf/acsos/Cortecchia24} as the reference implementation language,
due to its portability across platforms and its ergonomic library support for \ac{ac} programming.

To support collective formation and spatial organization,
we proposed a \emph{morphogenetic} approach to structure formation~\cite{DBLP:journals/comintsys/CortecchiaCCP26,DBLP:conf/acsos/CortecchiaPCC24}.
%
The approach leverages resource-flow management in a network of devices,
distributing resources towards the most successful area of the network in a self-organizing and self-adaptive way.
%
It is currently being generalized to generate resilient structures that follow mathematically defined shapes,
so that formations can be specified at the collective level and maintained under local failures.

Runtime adaptability has been investigated through a path-replanning mechanism~\cite{DBLP:conf/acsos/AguzziBCMPPV25}.
%
In this setting,
devices are assigned tasks that must be collectively executed while preserving the global objective.
%
When disruptions occur,
such as the failure of a device,
the swarm adapts its execution plan by redistributing the affected tasks among the remaining devices.

For distributed consensus,
we developed a self-stabilizing \emph{min-max gossip} algorithm~\cite{DBLP:conf/coordination/CortecchiaPV26}.
%
The algorithm provides an adaptive and resilient mechanism for reaching consensus on a selector,
such as the minimum or maximum value of sensed information distributed across the swarm.
%
It tolerates failures and topology changes without requiring explicit parametrization to reset the gossip memory.
%
This makes it a candidate primitive for communication and shared decisions between collective processes that occupy different spatial regions.

Collective sensing and monitoring have been addressed through \ac{dpf}~\cite{DBLP:conf/acsos/CortecchiaDCCV26},
an approach for state estimation in distributed systems.
%
By exploiting \ac{ac},
the proposed algorithms improve resilience to failures and support more accurate neighborhood estimation,
without requiring excessive information exchange among devices.

Safety during transient phases is the focus of ongoing work on runtime filtering.\footnote{\url{https://github.com/angelacorte/carol-experiments}}
%
This direction leverages \ac{cbf}~\cite{DBLP:conf/eucc/AmesCENST19} and \ac{clf}~\cite{DBLP:journals/ieeejas/LiWYWH23} theory applied to \ac{ac}~\cite{DBLP:conf/acsos/CortecchiaPD26}.
%
\ac{cbf} and \ac{clf} are used to define system-level constraints,
such as inter-robot distances and unsafe areas,
which are enforced while adaptive swarm-level objectives are computed.
%
This line of work directly targets the main safety limitation of self-stabilizing systems:
eventual convergence is not enough when transient states may already be unsafe.

Within the envisioned \ac{cros},
these mechanisms are intended to become reusable services for collective applications.

\section{Future Work and Research Plan}\label{sec:future-work-and-research-plan}

As next steps of the research,
the focus will be on the safety filter mechanism,
to improve it by defining constraints on swarm formation and runtime target-switching policies.
%
A follow-up of this work can be the integration of the \ac{cbf} within the morphogenesis algorithm,
to define the formation of more complex structures.

Moreover,
a particular focus will be on the support of preemption and the execution of multiple applications on the same device,
since this is a fundamental requirement for a \ac{cros} to be used in real-world scenarios.
%
Currently,
a general formal framework for the specification of collective computational processes has been proposed~\cite{EAAI2020-processes},
which are temporary distributed computations carried out by dynamic groups of devices,
as well as a set of algorithms for efficient propagation and termination of such concurrent execution processes~\cite{DBLP:journals/fgcs/AudritoCT24}.
%
However,
this approach still has some limitations on how a computational process should expand or contract across space,
so this will be investigated in the future to provide a more general approach to the execution of multiple applications on the same device.

Current contributions are not yet integrated into a unified framework;
therefore,
the final goal of this research is to define a prototype of a \ac{cros} based on \ac{ac} and Collektive,
focused on providing a set of reusable building blocks and algorithms that support the definition of global objectives and self-organization.
